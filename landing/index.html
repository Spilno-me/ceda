What is CEDA?

CEDA (Cognitive Event-Driven Architecture) is an AI-native copilot layer that transforms natural language input into structured module definitions through a multi-stage cognitive pipeline.

The system understands user intent, matches against domain-specific patterns, generates predictions, and validates outputs - all while learning from feedback to continuously improve.

● Generate structured modules from natural language
● Iteratively refine predictions through modification
● Apply domain-specific patterns (HSE, HR, Finance)
● Auto-validation and auto-correction of structures

↓
Structured Output

What is Herald MCP?

Herald MCP is the Model Context Protocol interface that connects AI assistants like Claude Desktop to CEDA's cognitive pipeline.

AI Assistant Integration

Connect Claude Desktop or other MCP-compatible AI assistants directly to CEDA for natural language module generation.

Dual-Mode Operation

Natural chat mode for humans, structured MCP protocol for AI agents. Same powerful cognitive pipeline, flexible interfaces.

Context Synchronization

Herald-to-Herald communication enables cross-domain context sharing and accumulated wisdom across projects.

MCP Tools

herald_predict Generate module structure from natural language
herald_refine Refine prediction with additional context
herald_health Check CEDA connection status
herald_session Get current session state
herald_accept Accept prediction, persist to session
herald_stats Get feedback statistics

Installation

Get started with Herald MCP in seconds using npx, or configure manually for more control.

Quick Install (Recommended)

Run the initialization command to automatically configure Herald MCP:

npx @spilno/herald-mcp init

This will guide you through setup and configure your MCP client automatically.

Manual Configuration

Add Herald MCP to your Claude Desktop configuration file:

Claude Desktop (~/.claude.json)

{
  "mcpServers": {
    "herald": {
      "command": "npx",
      "args": ["@spilno/herald-mcp"],
      "env": {
        "HERALD_API_URL": "https://getceda.com"
      }
    }
  }
}

Project-specific (.mcp.json)

{
  "mcpServers": {
    "herald": {
      "command": "npx",
      "args": ["@spilno/herald-mcp"],
      "env": {
        "HERALD_API_URL": "https://getceda.com",
        "HERALD_VAULT": "your-project"
      }
    }
  }
}

Global npm Install

Install Herald MCP globally for CLI access:

npm install -g @spilno/herald-mcp

Quick Start

Start generating structured modules in minutes.

1

Install Herald MCP

npx @spilno/herald-mcp init
2

Restart Claude Desktop

After configuration, restart Claude Desktop to load the Herald MCP server.

3

Start Creating

Ask Claude to generate modules using natural language:

You: "Create a safety assessment form with hazard identification and risk scoring"
Herald: Generates a complete module structure with sections, fields, and workflows
4

Refine & Iterate

Modify predictions with follow-up instructions:

You: "Add OSHA compliance checklist to the assessment"

CLI Commands

# Chat mode (interactive)
herald-mcp chat

# Command mode (structured)
herald-mcp predict "create safety assessment"
herald-mcp refine "add OSHA compliance"
herald-mcp health
herald-mcp stats
